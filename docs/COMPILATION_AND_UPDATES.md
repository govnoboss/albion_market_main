# Гайд по Компиляции и Обновлениям

## 1. Как работает компиляция (.exe)

Мы используем инструмент **Nuitka**. Это не просто "архиватор", как другие инструменты.
Процесс выглядит так:
1.  **Трансляция**: Nuitka берет ваш Python-код (`.py`) и переписывает его на язык **C++**.
2.  **Компиляция**: Затем запускается C-компилятор (GCC или MSVC), который превращает этот C++ код в **Машинный код** (нули и единицы, понятные процессору).
3.  **Сборка**: Все библиотеки (PyQt, Requests, OCR) упаковываются внутрь.

**Итог**: Получается один файл `GBot.exe`.
*   Внутри него **нет** исходного кода Python.
*   Его нельзя открыть "Блокнотом" или легким декомпилятором.
*   Он работает быстрее, так как это нативный код.

## 2. Как будет работать запуск

**Для пользователя:**
1.  Пользователь скачивает папку с ботом (там лежит `GBot.exe` и папка `resources/` с картинками).
2.  Дважды кликает по `GBot.exe`.
3.  Запускается программа.
    *   Пользователю **НЕ НУЖНО** устанавливать Python.
    *   Ему **НЕ НУЖНО** качать библиотеки (`pip install`).

**Что происходит под капотом:**
1.  Программа распаковывает нужные DLL в оперативную память.
2.  Проверяет файл лицензии (`.gbot_license` в папке или в профиле пользователя).
3.  Стучится на ваш Сервер.
4.  Если ОК -> Показывает Лаунчер.

## 3. Как делать обновления

Так как у нас нет встроенного авто-апдейтера (это сложная отдельная система), обновления делаются по классической схеме "Ручная замена":

**Ваши действия (как Разработчика):**
1.  Вы внесли изменения в код.
2.  Запускаете скрипт `build_gbot.bat`.
3.  Ждете 2-5 минут.
4.  Получаете новый `GBot.exe`.
5.  Заливате его на Google Drive / Mega / Ваш сайт.

**Действия пользователя:**
1.  Скачивает новый файл.
2.  Заменяет старый `GBot.exe` на новый.
3.  Запускает.
    *   Лицензия не слетает (она лежит в отдельном файле).
    *   Настройки не слетают.

Это самый надежный способ для старта.
